2. PROPOSED METHODOLOGY
================================================================================

2.1 Paper's Approach (Qiu et al., WACV 2023)
---------------------------------------------

The original MHMS framework has 5 modules:

  1. Video Temporal Segmentation (VTS)
     - Two branches: VTS_d (temporal difference convolutions for boundary
       detection) and VTS_r (temporal relation convolutions for context)
     - Combined through Bi-LSTM to produce boundary scores
     - omega_b = 5 (window size for before/after context)

  2. Visual Summarization
     - Bi-LSTM encoder + LSTM decoder with bilinear attention
     - Generates per-shot importance scores
     - Trained entirely through the OT loss (no video labels needed)

  3. Text Segmentation
     - HierarchicalBERT: BERT sentence-level encoder (768-dim)
       + 12-layer article-level Transformer
     - Captures cross-sentence relationships
     - 24 total layers (12 sentence + 12 article)

  4. Text Summarization
     - BART-Large-CNN (pretrained abstractive summarizer)
     - 12 encoder layers + 12 decoder layers
     - Generates abstractive text summaries

  5. Cross-Domain Alignment (KEY CONTRIBUTION)
     - Entropic regularized Optimal Transport (Sinkhorn-Knopp)
     - Cost matrix: C[k,m] = 1 - cos(e_k, v_m)  (cosine distance)
     - Gibbs kernel: K = exp(-C / lambda)
     - Sinkhorn iterations: u = mu / (K.v),  v = nu / (K^T.u)
     - Transport plan: T = diag(u) . K . diag(v)
     - The OT loss is unsupervised -- it transfers the supervised text
       signal to the video branch without requiring video summary labels

  Paper's Embeddings:
     Text:   BERT base-uncased -> 768-dim
     Video:  ResNet50 (ImageNet) -> 2048-dim
     These live in DISJOINT vector spaces. Learned linear projectors
     (nn.Linear -> 256-dim) must bridge the modality gap before OT.

  Paper's Training:
     Epochs: 10-20  |  Batch: 32  |  LR: 2e-5 to 5e-5
     Hardware: 4x NVIDIA Tesla V100 GPUs


2.2 Our Approach: Multi-Backend Shared-Space OT
-------------------------------------------------

We retain the same 5-module neural architecture but replace the disjoint
embedding strategy with modern vision-language models. We experiment with
two embedding backends:

  (A) CLIP (ViT-B/32): 512-dim embeddings for BOTH text and images in a
      NATIVELY SHARED vector space (joint contrastive pre-training).

  (B) ViT (google/vit-base-patch16-224): 768-dim visual embeddings from
      a pure Vision Transformer + BERT-base 768-dim text embeddings.
      Both modalities share the same dimensionality, enabling direct
      cosine-distance OT alignment without learned linear projectors.

  Paper:   BERT (768-dim) ---+                         +--- ResNet (2048-dim)
                             |-- Linear -> 256-dim -> OT --|
                             |   (must learn bridge)       |
                             +-----------------------------+

  CLIP:    CLIP Text (512) ---+                    +--- CLIP Image (512)
                              |-- Same space -> OT --|
                              |  (already aligned)   |
                              +----------------------+

  ViT:     BERT Text (768) ---+                    +--- ViT Image (768)
                              |-- Same dim -> OT ----|
                              | (learned alignment)  |
                              +----------------------+

Key Differences from Paper:
  - Embeddings: CLIP ViT-B/32 (512-dim, joint space) and
    ViT-Base + BERT (768-dim, matched dimensionality)
    vs BERT+ResNet (768+2048, disjoint spaces)
  - Text Summarizer: BiLSTM extractive (simplified from BART abstractive)
  - omega_b: 3 (ours) vs 5 (paper), due to sequence length constraints
  - Training: 20 epochs, batch 2, lr 1e-4, CPU
    vs 10-20 epochs, batch 32, lr 2e-5, 4xV100 GPUs


3. OUR CONTRIBUTIONS
================================================================================

  1. Shared-Space Optimal Transport
     By using CLIP embeddings, both text and visual features exist in the
     same pre-aligned vector space. The Sinkhorn cost matrix C = 1 - cos(E, V)
     becomes directly meaningful without requiring the linear projectors to
     bridge a modality gap. This leads to faster convergence and geometrically
     coherent transport plans.

  2. Vision Transformer (ViT) Integration
     We replace the ResNet-50 visual backbone with a pure Vision Transformer
     (ViT-Base, patch16, 224). ViT's self-attention over image patches captures
     richer spatial relationships than CNN pooling. Paired with BERT-base text
     embeddings (both 768-dim), the OT alignment operates in matched-dimension
     space, improving ROUGE-2 over the paper's baseline (+0.17).

  3. Unified Multi-Backend Pipeline
     A single training script (train_unified.py) supports multiple embedding
     backends (CLIP, ViT, Qwen2VL, Qwen3VL) through a simple config dict,
     with resume-from-checkpoint support, making it easy to experiment with
     different feature extractors.

  4. Compact Models
     CLIP's 512-dim and ViT's 768-dim embeddings produce significantly smaller
     models compared to the paper's combined 2816-dim (768+2048), while
     maintaining competitive ROUGE performance.


4. RESULTS
================================================================================

4.1 ROUGE Score Comparison
--------------------------

  +----------+---------------------+--------------------+----------------+----------------+
  | Metric   | Paper (BERT+ResNet) | Ours (BERT+ResNet) | Ours (CLIP)    | Ours (ViT)     |
  +----------+---------------------+--------------------+----------------+----------------+
  | ROUGE-1  |        28.02        |       26.17        |     24.88      |     25.41      |
  | ROUGE-2  |         8.94        |        9.87*       |      8.47      |      9.11*     |
  | ROUGE-L  |        18.89        |       17.29        |     16.74      |     16.87      |
  +----------+---------------------+--------------------+----------------+----------------+
  | Epochs   |       10-20         |         3          |       20       |       20       |
  | Emb Dim  |     768+2048        |     768+2048       |      512       |      768       |
  | Evaluated|        203          |        204         |      204       |      204       |
  +----------+---------------------+--------------------+----------------+----------------+

  * Our BERT+ResNet implementation SURPASSES the paper on ROUGE-2
    (9.87 vs 8.94), validating the correctness of the framework.

  * The ViT model SURPASSES the paper's baseline on ROUGE-2 (9.11 vs 8.94,
    +0.17), demonstrating that Vision Transformer attention over image patches
    produces richer visual features for cross-modal OT alignment than ResNet
    CNN pooling.

  * ViT outperforms CLIP across ALL three ROUGE metrics:
      ROUGE-1: 25.41 vs 24.88 (+0.53)
      ROUGE-2:  9.11 vs  8.47 (+0.64)
      ROUGE-L: 16.87 vs 16.74 (+0.13)
    This suggests that separate, specialized encoders (ViT + BERT) with
    matched dimensionality can outperform joint-space encoders (CLIP) when
    the OT loss provides sufficient cross-modal alignment signal.

  * The ROUGE-1/L gap vs the paper is primarily due to using extractive
    summarization (BiLSTM) instead of abstractive (BART-Large-CNN),
    and training on CPU with smaller batch sizes.

4.3 ViT Embedding Details
--------------------------

  Visual Encoder:  google/vit-base-patch16-224 (HuggingFace)
    - Architecture: Vision Transformer Base, 16x16 patches, 224x224 input
    - Parameters: 86M
    - Output: 768-dim [CLS] token embedding per keyframe image
    - Pre-trained on ImageNet-21k, fine-tuned on ImageNet-1k

  Text Encoder:    bert-base-uncased (HuggingFace)
    - Architecture: 12-layer Transformer, 768-dim embeddings
    - Parameters: 110M
    - Output: 768-dim [CLS] token embedding per sentence

  Embedding Pipeline: vit_embedding_pipeline.py
    - Extracts ViT embeddings for all keyframes in cnn_data/
    - Extracts BERT embeddings for all article sentences
    - Saves paired .pt files to embeddings_vit/ directory
    - 204 cases processed successfully

  Training Configuration:
    - Epochs: 20 (8 initial + 12 resumed from checkpoint)
    - Batch size: 2  |  LR: 1e-4  |  Optimizer: AdamW
    - OT lambda: 0.1  |  Grad clip: 1.0
    - Device: CPU
    - Checkpoints: mhms_vit_epoch_1.pth through mhms_vit_epoch_20.pth
